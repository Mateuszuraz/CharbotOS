Automatyzacja
OS File Agent
AI czyta i zapisuje pliki na Twoim dysku. Settings → OS Agent.

Wskaż dozwolone katalogi — agent nie wyjdzie poza nie. Poza tymi ścieżkami filesystem jest zablokowany.

Dostępne narzędzia

NarzędzieCo robi
list_dirListuje pliki i foldery w podanej ścieżce
search_filesSzuka plików po nazwie — max głębokość 4, max 50 wyników
read_fileCzyta plik tekstowy, CSV lub arkusz XLSX (max 500 KB)
write_fileTworzy lub nadpisuje plik
append_csvDopisuje wiersz do pliku CSV (tworzy z nagłówkami jeśli nie istnieje)

Jak używać

Pisz naturalnie — agent dobiera narzędzia sam:

Sprawdź co mam w ~/Downloads i posortuj pliki według rozmiaru.
Zapisz wynik do ~/Desktop/raport.txt

---

Otwórz ~/Dokumenty/wydatki.csv i policz sumę z kolumny "kwota".
Dopisz nowy wiersz z dzisiejszą datą i kwotą 250 zł.

Wyniki wywołań narzędzi widoczne są jako bąbelki TOOL CALL w czacie.

09
Rozszerzenia
Plugins
Własne narzędzia dla agenta. Settings → Plugins.
Typ A

Plik .mjs

Własny kod JavaScript. Pełna kontrola — wywołania API, przetwarzanie danych, dowolna logika.

Typ B

Webhook Tool

Bez kodowania. Podaj URL, metodę i parametry — Charbot wywołuje endpoint za Ciebie.

Typ A — plik .mjs

Umieść plik w CharbotVault/plugins/ i kliknij Reload:

// CharbotVault/plugins/kurs_btc.mjs
export const name = 'kurs_btc';
export const description = 'Pobiera aktualny kurs BTC w PLN.';
export const parameters = {};

export async function execute(_args, _ctx) {
  const r = await fetch(
    'https://api.coingecko.com/api/v3/simple/price?ids=bitcoin&vs_currencies=pln'
  );
  const d = await r.json();
  return `BTC: ${d.bitcoin.pln} PLN`;
}

PolePrzykład

Privacy First
Bezpieczeństwo
Charbot OS projektowany jest z myślą o prywatności od podstaw.
P0

Lokalny bind

Serwer nasłuchuje wyłącznie na 127.0.0.1 — niedostępny z zewnątrz sieci.

P0

APP_SECRET

Każde żądanie API wymaga tokenu generowanego przy starcie. Brak tokenu = 401.

P0

Vault Sandbox

OS Agent działa tylko w katalogach, które sam wskazałeś. Zero dostępu poza nimi.

P0

Klucze w Vault

Klucze API w CharbotVault/ai-keys.json. Nigdy w przeglądarce, nigdy w localStorage.

P1

Offline Mode

CHARBOT_OFFLINE=true blokuje wszystkich dostawców chmurowych. Tylko Ollama.

P1

Rate Limiting

60 req/min na /api, 40 req/min na endpointy AI. Ochrona przed atakami.

Zmienne środowiskowe

# Lokalizacja Vault (domyślnie ~/CharbotVault)
CHARBOT_VAULT_DIR=D:/MójVault

# Zezwól na wykonywanie poleceń shell przez AI

# Zezwól na fine-tuning modeli Ollama

# Tryb uncensored (brak cenzury w AI)
CHARBOT_UNCENSORED=true

# Bind na interfejsie sieciowym (domyślnie 127.0.0.1)
# Zmień na 0.0.0.0 aby udostępnić w sieci LAN
BIND_HOST=0.0.0.0

# Własny token autoryzacji (generowany automatycznie jeśli brak)
APP_SECRET=mój_tajny_token

# Tokeny botów
DISCORD_BOT_TOKEN=twój_token

Struktura Vault
Wszystkie dane aplikacji w jednym folderze — łatwy backup.
CharbotVault/ ├── charbot.db ← baza SQLite (sesje, scheduler, RAG) ├── ai-keys.json ← klucze API (tylko lokalnie) ├── app-secret.txt ← token autoryzacji (autogenerowany) ├── telegram-config.json ← konfiguracja bota Telegram ├── plugins-webhook.json ← webhook tools ├── uploads/ ← przesłane pliki (per sesja) ├── documents/ ← zapisane artefakty AI ├── plugins/ ← wtyczki .mjs ├── mobile/ ← portal mobilny (HTML) └── logs/ ← logi serwera (rotowane co 5 MB)
💡
Zrób backup całego folderu CharbotVault/ — to wszystko czego potrzebujesz. Baza danych, klucze, pliki — wszystko w jednym miejscu.

Nowość · v2.2
Multiroom — Wspólna praca z AI
Wiele osób, wiele modeli, jeden pokój. Real-time.

Multiroom pozwala kilku użytkownikom pracować jednocześnie w tym samym czacie — każdy ze swoim modelem AI. Wiadomości i odpowiedzi AI są synchronizowane w czasie rzeczywistym przez WebSocket. Idealne do code review, burzy mózgów i porównywania odpowiedzi różnych modeli.

Jak to działa technicznie

01
Twórca otwiera pokój
02
Wysyła link zaproszenia
03
Każdy dołącza z username + model
04
WebSocket: tokeny AI w czasie rzeczywistym
05
Historia zapisana w SQLite
🔌
Każdy uczestnik łączy się przez WebSocket na /ws/room/:id. Serwer streamuje tokeny AI (ai_token) do wszystkich jednocześnie — każdy widzi odpowiedź modelu dosłownie litera po literze.

Cztery tryby AI

Tryb 1

Własny model

Każdy uczestnik ma przypisany model. AI odpowiada wyłącznie na wiadomości swojego użytkownika. Naturalne, konwersacyjne środowisko.

Tryb 2

Panel AI

Każda wiadomość → wszystkie modele w pokoju odpowiadają kolejno. Idealne do porównywania odpowiedzi: Llama3 vs Mistral vs Gemma na to samo pytanie.

Tryb 3

@mention

Tylko wywołany model odpowiada. Składnia: @llama3 napisz mi testy jednostkowe. Pełna kontrola nad tym, który model reaguje.

Tryb 4

Debata

Dwa modele dyskutują z przeciwnych stron tezy. Model PRO argumentuje za, model CON — przeciw. Ty moderujesz, podając kolejne tezy.

Krok po kroku — tworzenie pokoju

01

Otwórz panel boczny → tab "Pokoje"

W lewym panelu sesji kliknij zakładkę Pokoje (ikona ludzi). Widoczna jest lista aktywnych pokojów.

02

Kliknij "Nowy pokój"

Wpisz nazwę, wybierz tryb AI (własny model / panel / @mention / debata). Opcjonalnie ustaw hasło i instrukcje debaty.

03

Dołącz — wpisz username i model

Podaj swoją nazwę widoczną dla innych (np. Mateusz) oraz model Ollama, który chcesz używać (np. llama3.2).

04

Udostępnij link

Przycisk "Kopiuj link" w nagłówku pokoju kopiuje URL do schowka. Wyślij go do innych uczestników — otwierają ten sam Charbot OS w sieci lokalnej (http://IP:3000).

05

Pisz — AI odpowiada wszystkim jednocześnie

Wiadomości i tokeny AI są widoczne u wszystkich w czasie rzeczywistym. Wskaźnik połączenia (zielony/żółty/czerwony) pokazuje stan WebSocket.

Interfejs pokoju

ElementCo robi
NagłówekNazwa pokoju, badge trybu, liczba online, wskaźnik WS, przyciski Kopiuj link / Opuść
Lista wiadomościKolorowe awatary uczestników, badge modelu AI, streaming tokenów z animowanym kursorem

Pole wpisywaniaEnter = wyślij, Shift+Enter = nowa linia. Informacja o aktywnym modelu pod polem.
